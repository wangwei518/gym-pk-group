# 微信好友运动打卡PK小程序产品需求文档（PRS）
 - 适用平台：微信小程序
 - 后端方案：微信云开发（免费版数据库）
 - 运动项目：引体向上 / 跳绳 / 攀岩 / 跑步

## 1. 项目概述
### 1.1 项目背景
为提升用户日常运动积极性，通过好友间运动数据排行与PK形成正向激励，开发一款轻量化运动打卡小程序。用户可每日进行多项目、多次运动打卡，并在主页面查看同项目好友累计排名与进度对比

### 1.2 产品目标
- 实现微信一键登录，无需注册
- 支持引体向上、跳绳、攀岩、跑步四类运动打卡
- 每日可打卡多种项目，同一项目支持多次打卡
- 主页面可切换项目，展示好友累计数值排行
- 数据存储使用微信云开发免费数据库，零成本、稳定、数据不丢失
- 界面简洁、操作简单、BUG少、体验流畅

### 1.3 用户群体
- 有固定运动习惯的用户
- 希望与好友互相监督、比拼运动数据的用户
- 偏好轻量化工具，不想安装独立App的用户

## 2. 页面结构
- 布局(上下布局)
  - 主页面
    - 排行榜（排行榜）
    - 个人页面（我的）
  - 底部Tab（切换主页面结构）
- 交互
  - 同时只显示排行榜和个人页面其中以一个，用底部tab切换
  - 底部tab选中的页面，底色高亮

### 2.1 排行榜（排行榜）
- 布局（上下布局）
  - 次级Tab，切换运动项目
    - 引体向上
    - 跳绳
    - 攀岩
    - 跑步
  - 排行榜列表(显示条目，按当前项目累计总数降序排列)
    - 条目（用户头像+用户昵称+进度条+右侧显示该项目累计总数及单位）
  - 悬浮球（手动打卡入口，后台选中当前项目，用户可填写次数或公里数）

### 2.2 个人页面（我的）
- 布局
  - 用户头像
  - 昵称
  - 运动数据概览：三项运动各自累计总数
  - 历史打卡记录列表（按时间倒序）

## 3. 功能需求
### 3.1 用户登录与信息
- 微信静默授权登录，自动获取openid、昵称、头像
- 首次进入自动创建用户信息并保存至云数据库
- 用户信息全局唯一，更换设备不丢失

### 3.2 运动打卡
- 支持项目：
  - 引体向上（单位：次）
  - 跳绳（单位：次）
  - 攀岩（单位：次）
  - 跑步 (单位：公里)
- 每日可打卡多个不同项目
- 同一项目一天内允许多次打卡，次数自动累加
- 每次打卡记录：项目类型、本次数量、打卡时间
- 打卡成功后实时同步至云端，排行榜立即更新

### 3.3 排行榜
- 顶部Tab切换运动项目，内容实时刷新
- 按项目展示所有用户累计数据排行
- 进度条长度与第一名数值成比例
- 每条记录右侧显示精确总次数/总公里数
- 数据来源：微信云开发数据库
- 提供快速打卡入口，支持一键提交

### 3.4 个人中心
- 展示本人三项运动总数据
- 展示历史打卡记录

## 4. 数据与后端方案
### 4.1 后端方案
- 数据库：微信云开发 - 免费版云数据库
- 无需自建服务器
- 无需域名、无需备案
- 免费额度可完全满足本项目正常运行

### 4.2 数据库集合设计
#### users（用户表）
| 字段名       | 说明                     |
|--------------|--------------------------|
| openid       | 微信用户唯一标识         |
| nickName     | 昵称                     |
| avatarUrl    | 头像                     |
| createTime   | 创建时间                 |
| updateTime   | 更新时间                 |

#### records（打卡记录表）
| 字段名       | 说明                     |
|--------------|--------------------------|
| _id          | 系统主键                 |
| openid       | 用户标识                 |
| type         | 项目类型 pullup/rope/rock/run|
| value        | 本次打卡次数             |
| createTime   | 打卡时间                 |
| date         | 日期字符串 如 2025-04-18 |

### 4.3 免费额度说明
- 数据库存储：1GB
- 读请求：100万次/月
- 写请求：5万次/月
- 云函数调用：100万次/月
- 本项目为轻量打卡工具，永久在免费额度内运行

## 5. 非功能需求
### 5.1 性能
- 小程序冷启动 ≤ 3 秒
- 项目Tab切换响应 ≤ 1 秒
- 排行榜加载完成 ≤ 2 秒
- 打卡提交并同步 ≤ 2 秒

### 5.2 稳定性
- 核心功能（登录、打卡、排行）可用率 ≥ 99.9%
- 网络异常时自动重试并友好提示
- 无闪退、无卡死、无数据错乱

### 5.3 兼容性
- 微信基础库 ≥ 2.10.0
- 兼容iOS / Android主流机型
- 适配常规全面屏、刘海屏、挖孔屏

### 5.4 成本
- 全程使用微信云开发免费资源
- 无服务器费用、无数据库费用、无流量费用

## 6. 开发技术栈
- 前端：原生微信小程序 + TypeScript
- UI：官方基础组件 + 自定义排行列表、进度条
- 存储：微信云开发云数据库
- 逻辑：轻量化封装，结构清晰、易维护、BUG少

## 7. 开发阶段
1. 环境搭建（1～2 天）
   - 新建小程序项目
   - 开通并初始化云开发环境
   - 创建数据库集合与权限配置

2. 页面结构开发（2 天）
   - 底部Tab框架
   - 主页面顶部项目Tab
   - 排行榜列表UI

3. 核心功能开发（3～5 天）
   - 微信登录与用户信息同步
   - 打卡功能与数据上传
   - 排行榜统计、排序、进度条展示

4. 联调与测试（2～3 天）
   - 多用户数据互通测试
   - 多次打卡累加逻辑测试
   - 云开发环境稳定性测试
   - BUG修复与体验优化

## 8. 验收标准
1. 可正常完成微信授权登录
2. 三个运动项目可正常切换，排行数据正确
3. 多项目、多次打卡次数累加正确
4. 主页面进度条 + 右侧数字展示正常
5. 所有数据存储在微信云开发免费数据库
6. 更换手机/重装小程序后数据不丢失
7. 界面流畅、无明显Bug、符合需求描述
